① Dictionary — Was ist das?

Bisher kanntest du Listen: ["Pizza", "Burger"]. Aber was, wenn du zu jedem Essen den Preis speichern willst?

Liste vs Dictionary

# Liste: Zugriff über Position (Index)
preise = [8.50, 6.00, 3.50]
print(preise[0])          # 8.50 — du musst WISSEN, dass Pizza an Position 0 steht

# Dictionary: Zugriff über NAME (Key)
preise = {"Pizza": 8.50, "Burger": 6.00, "Pommes": 3.50}
print(preise["Pizza"])      # 8.50 — klar und lesbar!

Die Idee: Schlüssel → Wert

Schlüssel (Key) [键]Wert (Value) [值]
"Pizza"8.50
"Burger"6.00
"Pommes"3.50

Wie ein Wörterbuch: du schlägst ein Wort (Key) nach → bekommst die Bedeutung (Value).

🍪 Wissens-Snack: In anderen Sprachen heißt das HashMap, Hashtable oder assoziatives Array. Gleiche Idee: "Gib mir den Wert zu diesem Namen." Intern extrem schnell — fast egal wie groß das Dict ist.

② Dict-Operationen

# Erstellen
menü = {"Pizza": 8.50, "Burger": 6.00}
# Zugriff
print(menü["Pizza"])           # 8.50
# Hinzufügen / Ändern
menü["Salat"] = 5.00            # neu hinzu
menü["Pizza"] = 9.00            # überschreiben
# Löschen
del menü["Burger"]
# Prüfen ob Key existiert
if "Pizza" in menü:
    print("Pizza ist auf der Karte!")
# Durchiterieren
for name in menü:
    print(f"{name} kostet {menü[name]}€")
# Keys & Values getrennt
print(list(menü.keys()))       # ['Pizza', 'Salat']
print(list(menü.values()))     # [9.00, 5.00]

Warum Dicts so wichtig sind

Fast alles in Python basiert auf Dicts — Variablen, Module, sogar Klassen sind intern Dicts. Sie sind die universelle Datenstruktur der Sprache.

🍪 Wissens-Snack: Python's eigene Variablen leben in einem Dict! globals() gibt dir ALLE globalen Variablen als Dict zurück. Probier mal im VSCode: print(globals()) — du siehst deine eigenen Funktionen und Variablen darin!

③ JSON — die Sprache des Internets

JSON = JavaScript Object Notation. Ist im Prinzip ein Python-Dict in Textform.

Python-Dict → JSON-String → und zurück

import json

# Dict → JSON-String (dumps = dump string)
menü = {"Pizza": 8.50, "Burger": 6.00}
text = json.dumps(menü, ensure_ascii=False)
print(text)  # {"Pizza": 8.5, "Burger": 6.0}

# JSON-String → Dict (loads = load string)
daten = json.loads(text)
print(daten["Pizza"])  # 8.5

# In Datei speichern
with open("menü.json", "w") as f:
    json.dump(menü, f, ensure_ascii=False)

# Aus Datei laden
with open("menü.json", "r") as f:
    geladen = json.load(f)
FunktionRichtung
json.dumps(obj)Python-Objekt → String [序列化]
json.loads(text)String → Python-Objekt [反序列化]
json.dump(obj, file)Python-Objekt → Datei
json.load(file)Datei → Python-Objekt
🍪 Wissens-Snack: Warum ensure_ascii=False? Ohne das wird "Pizza" zu "Pizza" aber "Überraschung" würde zu "\u00dcberraschung" — unlesbar! ensure_ascii=False sagt: "Schreib deutsche Buchstaben direkt, nicht als Code."

④ Was ist eine API?

API = Application Programming Interface [接口].

Stell dir ein Restaurant vor:
Du (dein Programm) sitzt am Tisch. Du willst Essen.
Die Speisekarte = API-Dokumentation
Der Kellner = die API
Du sagst dem Kellner "Pizza Margherita" (das ist ein Request).
Der Kellner geht in die Küche, kommt zurück mit einer Pizza (Response).
Du musst nicht wissen, wie die Küche funktioniert.

Echte APIs — dein erster Request

# Erstmal pip installieren: pip3 install requests
import requests

# Eine echte API abfragen (gibt einen Witz zurück)
antwort = requests.get("https://v2.jokeapi.dev/joke/Programming?lang=de")
daten = antwort.json()              # Antwort als Dict!

if daten["type"] == "single":
    print(daten["joke"])
else:
    print(daten["setup"])
    print(daten["delivery"])
💡 Das ist der große Moment: Du holst Daten aus dem INTERNET in dein Python-Programm. Die API schickt JSON, Python macht ein Dict draus — und du arbeitest damit wie mit jedem anderen Dict. Genau so funktionieren Wetter-Apps, Chat-Programme, dein Studienhelfer — alles.

Die Struktur ist IMMER gleich:

Dein Programm → Request (URL) → API-Server → Response (JSON) → Dein Programm als Dict

⑥ Aufgabe: Vokabeltrainer

Bau einen einfachen Deutsch-Englisch Vokabeltrainer:

  1. Ein Dict mit 5 Vokabeln: {"Haus": "house", "Katze": "cat", ...}
  2. Speicher das Dict als vokabeln.json
  3. Programm fragt: "Was heißt 'Haus' auf Englisch?"
  4. Prüft die Antwort, zählt richtige/falsche
  5. Am Ende: "3/5 richtig!"

Gerüst:

import json

vokabeln = {
    "Haus": "house",
    "Katze": "cat",
    "Hund": "dog",
    "Buch": "book",
    "Schule": "school"
}

# In JSON speichern
with open("vokabeln.json", "w") as f:
    json.dump(vokabeln, f, ensure_ascii=False)

# Abfragen
richtig = 0
for de, en in vokabeln.items():         # .items() = beide, Key UND Value
    antwort = input(f"Was heißt '{de}' auf Englisch? ")
    if antwort.strip().lower() == en:
        print("✅ Richtig!")
        richtig += 1
    else:
        print(f"❌ Falsch! Richtig ist: {en}")

print(f"\n{richtig}/{len(vokabeln)} richtig!")
